## 一、项目愿景

### 最终目标
打造一个**快消品经销商一体化管理系统**，实现：
- **老板**：PC端掌控全局，手机端功能完全同步，随时随地就是办公室
- **业务员**：手机搞定8件事——待办/拜访/打卡/下单/打印/业绩/查库存/应收
- **终端店老板**：微信H5直接下单，不用装App不用注册
- **Agent机器人**：微信客服+推销+接单+查库存，24小时在线

### 核心业务流程（全链路）

```
┌──────────────────────────────────────────────────────────┐
│ ① Agent推销         ② 终端下单         ③ 老板审核       │
│ ──────────         ──────────         ──────────        │
│ Agent在微信群/      终端老板微信H5      老板PC/手机端     │
│ 私聊主动推销        直接下单            查看订单详情       │
│ 发促销朋友圈        加入购物车          核对价格/库存     │
│ 自动回复询价        确认下单            一键审核通过       │
│                     系统自动生成        分配业务员         │
│                     SO销售订单          设定信用额度       │
│      │                  │                  │             │
│      └──────────┬───────┴──────────┬───────┘             │
│                 ↓                  ↓                     │
│ ④ 业务员处理        ⑤ 仓库出库         ⑥ 送货收款        │
│ ──────────         ──────────         ──────────        │
│ 手机收到待办        仓管PC端扫码        业务员送货上门     │
│ 查看订单明细        确认商品+数量       终端老板收货       │
│ 按路线拜访          蓝牙打印销售单      手机端确认签收     │
│ 门店打卡签到        库存自动扣减        现场收款/挂账     │
│ 代客下单补货        应收自动增加        打印收款小票       │
│      │                  │                  │             │
│      └──────────────────┴──────────────────┘             │
│                         ↓                                │
│ ⑦ 财务闭环：应收账龄管理 → 催款提醒 → 对账单分享 → 结清  │
└──────────────────────────────────────────────────────────┘
```

**每个阶段的关键数据流转：**

| 阶段 | 谁做 | 用什么 | 输出什么 | 谁看到 |
|------|------|--------|----------|--------|
| ① Agent推销 | Agent机器人 | 微信私聊/群/朋友圈 | 客户询问/兴趣 | 终端老板 |
| ② 终端下单 | 终端店老板 | 微信H5页面 | SO销售订单 | 老板+业务员 |
| ③ 老板审核 | 老板 | PC端/手机端 | 审核通过+分配 | 业务员+仓管 |
| ④ 业务员处理 | 业务员 | 手机H5 | 拜访记录/补货单 | 老板+仓管 |
| ⑤ 仓库出库 | 仓管 | PC端/网页端 | SOUT出库单 | 业务员+财务 |
| ⑥ 送货收款 | 业务员 | 手机H5 | 签收+收款单RC | 财务 |
| ⑦ 财务闭环 | 财务 | PC端 | 对账/催款/核销 | 老板 |

## 二、三端规划

### 2.1 PC桌面端（Electron）
**用户**：老板、文员、财务、仓管
**定位**：全功能管理后台

| 功能模块 | 说明 |
|---------|------|
| 系统初始化 | 公司信息/仓库/员工/权限/打印模板/期初数据 |
| 档案管理 | 商品/客户/供应商档案 + 批量导入 |
| 采购管理 | 订单→入库→退货→发票→对账→统计 |
| 销售管理 | 订单→出库→退货→发票→对账→统计 |
| 仓库管理 | 实时库存/盘点/调拨/报损报溢/预警 |
| 财务管理 | 收款/付款/预收预付/应收应付/发票/账户余额 |
| 报表分析 | 经营看板/销售报表/采购报表/库存报表/利润表 |
| 客户管理 | 联系人/拜访记录/价格等级/流失预警 |
| 业务员管理 | 业绩统计/提成计算/拜访路线 |
| 辅助功能 | 数据备份/导入导出/操作日志/消息提醒 |
| **打印管理** | 销售单/采购单/入库单/对账单/快递单打印 |

### 2.2 手机端（手机网页/H5）

> **核心理念**：同一个后端API，不同前端呈现。老板手机端=PC端全功能适配手机屏幕，业务员手机端=8个极简窗口。

#### 老板手机端（功能完全等同于PC端）

**一句话：老板出差/在家/在外，打开手机就等同于打开电脑。**

老板手机端不裁功能，而是通过以下方式适配手机：

| PC端模块 | 手机端呈现方式 |
|----------|---------------|
| 经营看板 | 首页卡片式，今日销售额/回款/订单数/预警红点 |
| 报表分析 | 折叠菜单，点击展开；图表自适应竖屏 |
| 订单管理 | 列表滑动，点击进详情，支持审核/驳回 |
| 库存查询 | 搜索框+分类筛选，点击看流水 |
| 客户管理 | 客户列表+搜索，点击看详情/应收/拜访记录 |
| 财务管理 | 应收应付列表，账龄颜色区分 |
| 采购管理 | 订单列表，支持入库确认 |
| 消息提醒 | 顶部通知栏，库存预警/应收超期实时推送 |

**适配原则**：
- 所有数据接口和PC端完全一致（同一套API）
- 布局：卡片式代替表格，折叠菜单代替侧边栏
- 操作：长按=右键菜单，滑动=翻页
- 颜色语义统一：绿色=收入/完成，红色=支出/逾期，橙色=预警

### 2.3 微信生态

#### 终端店老板H5下单（微信内直接打开）

> **核心理念**：终端老板不用装App、不用注册账号，在微信里打开链接就能下单。

**下单流程（极致简化）：**

```
微信打开H5链接
    ↓
① 手机号验证（首次自动注册为客户）
    ↓
② 进入商品列表页
   - 显示该客户等级价（VIP/A/B/C）
   - 分类浏览：饮料/零食/日用品/...
   - 搜索框：搜商品名或扫条码
    ↓
③ 点商品 →
   - 显示价格+库存（只看总仓库存）
   - [+] [-] 选数量
   - 自动加入购物车
    ↓
④ 购物车 → 点击结算
   - 显示总金额
   - 显示当前欠款
   - 收货地址（默认上次地址）
    ↓
⑤ 确认下单
   - 提交 → 系统生成SO销售订单
   - 推送给老板审核
   - 推送通知给对应业务员
    ↓
⑥ 订单状态追踪（在H5页面查看）
   - 待审核 → 已审核 → 已出库 → 配送中 → 已送达
```

**H5页面功能清单：**

| 功能 | 说明 |
|------|------|
| 商品浏览 | 分类+搜索，显示客户专属价格 |
| 购物车 | 加减商品，一键下单 |
| 订单历史 | 查看历史订单+状态追踪 |
| 欠款查询 | 当前欠款总额+明细 |
| 对账单 | 查看/下载月度对账单 |
| 促销专区 | 显示当前优惠活动，限时标红 |
| 联系业务员 | 一键拨打业务员电话 |
| 确认收货 | 收货后点击确认，完结订单 |

**技术方案**：
- 前端：Vue3 + Vant UI（移动端组件库），打包为H5单页应用
- 部署：挂到服务器上，生成链接，微信内直接打开
- 认证：手机号+验证码，自动匹配客户档案中的手机号
- 分享：支持"帮我下单"链接分享（带客户ID参数）

#### Agent微信客服机器人

> **核心理念**：Agent作为24小时在线的虚拟业务员，主动推销+被动接单+智能客服。

**三大场景：**

##### 场景一：主动推销 📢

```
方式1：朋友圈推广
Agent自动发朋友圈：
  "🔥 康师傅冰红茶特价！整箱只要¥60！
   限时3天，手慢无！联系我下单→"
  [配商品图片]

方式2：群发促销
按客户标签群发：
  - 标签：商超客户 → 推整箱装
  - 标签：便利店 → 推小包装
  - 标签：欠款客户 → 催款提醒
  - 标签：流失客户 → 优惠券召回

方式3：微信群推销
在客户群中发布：
  "各位老板早上好！今日特价：统一冰红茶
  批发价¥2.5/瓶，武昌仓现货120箱！
  需要的老板直接@我下单～"
```

##### 场景二：智能接单 🤖

```
客户私聊Agent：
"我要5箱冰红茶"

Agent自动处理流程：
① 识别意图：下单
② 匹配商品：统一冰红茶500ml×24瓶
③ 确认客户：好邻居便利店（匹配微信→客户档案）
④ 确认价格：等级价 ¥2.5/瓶，5箱=¥300
⑤ 回复确认：
   "好的老板！统一冰红茶500ml，
   5箱×24瓶=120瓶，批发价¥2.5/瓶，
   合计¥300。
   现在就帮您下单？
   回复「确认」即可下单～"
⑥ 客户回复"确认" → Agent自动生成SO订单
⑦ 推送通知给老板审核

复杂意图也支持：
"上次那个冰红茶还有吗？再来5箱，顺便带2箱方便面"
→ Agent解析：冰红茶5箱+方便面2箱，一次性生成订单
```

```

客户：我欠多少钱
Agent：您当前欠款¥1,560，其中超期15天，
      建议尽快结清哦～我可以帮您查明细！

客户：冰红茶有货吗
Agent：有的！武昌总仓120箱、汉口48箱、汉阳15箱。
      需要多少？现在下单明天送到！

客户：什么时间送货
Agent：上午10点前下单，当天下午送到；
      10点后下单，次日送达。
      武昌/汉口/汉阳区域均支持配送！
```

**Agent技术实现：**

| 组件 | 方案 |
|------|------|
| 微信接入 | 企业微信API / 微信客服消息接口 |
| 意图识别 | 关键词匹配 + 大模型（DeepSeek/Hermes） |
| 商品匹配 | 模糊搜索匹配商品档案库 |
| 客户匹配 | 微信open_id → 客户档案手机号 |
| 订单生成 | API调用后端生成SO订单 |
| 自动回复 | 模板生成 + AI润色 |

**Agent安全边界：**
- 不允许自主修改价格（调用客户等级价）
- 不允许修改库存数据（只读查询）
- 生成订单需要老板审核（大额/新客户必审）
- 敏感操作（删单/退款）过滤，转人工

## 三、打印功能专章

> **核心理念**：PC端打印机打印大单，手机蓝牙打印小票，两种场景都支持。

### 3.1 打印场景总览

| 场景 | 谁打印 | 打印什么 | 用什么设备 | 打印方式 |
|------|--------|----------|------------|----------|
| 仓库出库 | 仓管 | 销售单/出库单（A4） | PC端打印机 | 浏览器打印/PDF |
| 开票对账 | 财务 | 对账单/发票 | PC端打印机 | PDF打印 |
| 外勤送货 | 业务员 | 销售单+小票 | 手机蓝牙打印机 | 蓝牙热敏打印 |
| 收款凭证 | 业务员 | 收款小票 | 手机蓝牙打印机 | 蓝牙热敏打印 |
| 客户现场 | 业务员 | 订单小票（给客户留底） | 手机蓝牙打印机 | 蓝牙热敏打印 |
| 仓库标签 | 仓管 | 商品标签/保质期标 | 蓝牙标签打印机 | 蓝牙热敏打印 |

### 3.2 PC端打印方案

```
PC端打印流程：
选择单据 → 预览 → 选择模板 → 打印/导出PDF
```

**打印模板类型：**

| 模板 | 纸张 | 内容 |
|------|------|------|
| 销售单 | A4 | 公司抬头+客户信息+商品明细(品名/规格/数量/单价/金额)+合计+签字区 |
| 采购单 | A4 | 供应商信息+商品明细+签字区 |
| 入库单 | A4 | 仓库+供应商+商品明细 |
| 出库单 | A4 | 仓库+客户+商品明细 |
| 对账单 | A4 | 客户+期初+本期发生+期末+账龄 |
| 快递单 | 快递面单 | 收件人信息，对接快递鸟API |

**技术方案：**
- PC端（Electron）：jsPDF生成PDF → 浏览器打印API
- 网页端：jsPDF生成PDF → 浏览器打印
- 模板可自定义：公司Logo/地址/电话/银行账号
- 支持批量打印（选中多单一次性打印）

### 3.3 手机端蓝牙打印方案

```
手机端蓝牙连接流程：
打开打印窗口 → 搜索蓝牙设备 → 连接打印机 → 选择订单 → 打印
```

**支持的蓝牙打印机：**

- 自动记住上次连接的设备

### 模块三：采购管理

#### 3.1 采购订单
- 关联供应商，自动带出商品
- 数量/单价/金额自动计算
- 状态：待审核→已审核→已入库→已关闭

#### 3.2 采购入库
- 关联或不关联采购订单
- 扫码入库（扫码枪）
- **估价入库**（货先到票未到，先估价入账）
- 确认入库 → 更新库存+供应商应付+成本价

#### 3.3 采购退货
- 关联原入库单
- 退货原因：质量/发错/过期
- 确认退货 → 库存减少+供应商应付减少

#### 3.4 采购发票
- 关联入库单
- 发票类型：专票/普票/收据
- 发票状态：未收/已收/已认证

#### 3.5 供应商对账
- 期初应付 + 本期采购 - 本期退货 - 本期付款 = 期末应付
- 明细可展开
- 打印对账单（客户签字）
- **支持微信分享对账单给供应商**

#### 3.6 采购统计
- 按供应商/商品/日期统计
- 趋势图：日/周/月/年
- 同比环比分析

---

### 模块四：销售管理

#### 4.1 销售订单
- 关联客户，自动带出等级价/专属价
- 关联业务员
- **整单折扣+单品折扣**
- **最低售价检查**（低于成本弹窗）
- **信用额度检查**（超额弹窗）
- **来源标记**：微信H5下单 / Agent代客下单 / 业务员手机下单 / PC端下单
- 状态：待审核→已审核→已出库→已关闭

#### 4.2 销售出库
- 关联或不关联销售订单
- **扫码出库**（扫码枪）
- 确认出库 → 库存减少+客户应收增加

#### 4.3 销售退货
- 关联原出库单
- **退货处理**：退回入库（可销售）/ 报损（不可销售）
- 确认退货 → 库存+（看处理方式）+客户应收减少

#### 4.4 销售发票
- 关联出库单
- 发票类型：专票/普票/收据

#### 4.5 客户对账
- 期初应收 + 本期销售 - 本期退货 - 本期收款 = 期末应收
- **超期催款提醒**（Agent自动微信催款）
- **微信分享对账单给客户**（PDF链接）
- 打印催款单

#### 4.6 销售统计
- 按客户/商品/业务员/日期统计
- 毛利率分析
- 趋势图

---

### 模块五：仓库管理

#### 5.1 实时库存
- 按仓库/按商品查询
- 库存金额（移动加权平均成本）
- **库存流水**：每个商品的完整出入库记录
- 导出Excel

#### 5.2 库存预警

| 预警类型 | 说明 |
|---------|------|
| 库存下限 | 低于设置值，首页+微信提醒 |
| 库存上限 | 超过设置值，避免积压 |
| 临期预警 | 保质期快到了（可设N天提前量） |
| 滞销预警 | N天无销售，标记滞销商品 |

#### 5.3 库存盘点
- 全盘/按分类/按商品
- 账面vs实盘，自动计算差异
- 确认 → 自动生成盘盈入库单或盘亏出库单

#### 5.4 库存调拨
- 调出仓→调入仓
- 同仓校验（不能自己调自己）

#### 5.5 报损报溢
- 报损：过期/破损 → 库存减少
- 报溢：盘点多 → 库存增加

#### 5.6 其他出入库
- 非采购/销售的出入库（赠品/样品/内部领用）

### 模块六：财务管理

#### 6.1 收款登记
