## 一、项目愿景

### 最终目标
打造一个**快消品经销商一体化管理系统**，实现：
- **老板**：PC端掌控全局，手机端功能完全同步，随时随地就是办公室
- **业务员**：手机搞定8件事——待办/拜访/打卡/下单/打印/业绩/查库存/应收
- **终端店老板**：微信H5直接下单，不用装App不用注册
- **Agent机器人**：微信客服+推销+接单+查库存，24小时在线

### 核心业务流程（全链路）

```
┌──────────────────────────────────────────────────────────┐
│ ① Agent推销         ② 终端下单         ③ 老板审核       │
│ ──────────         ──────────         ──────────        │
│ Agent在微信群/      终端老板微信H5      老板PC/手机端     │
│ 私聊主动推销        直接下单            查看订单详情       │
│ 发促销朋友圈        加入购物车          核对价格/库存     │
│ 自动回复询价        确认下单            一键审核通过       │
│                     系统自动生成        分配业务员         │
│                     SO销售订单          设定信用额度       │
│      │                  │                  │             │
│      └──────────┬───────┴──────────┬───────┘             │
│                 ↓                  ↓                     │
│ ④ 业务员处理        ⑤ 仓库出库         ⑥ 送货收款        │
│ ──────────         ──────────         ──────────        │
│ 手机收到待办        仓管PC端扫码        业务员送货上门     │
│ 查看订单明细        确认商品+数量       终端老板收货       │
│ 按路线拜访          蓝牙打印销售单      手机端确认签收     │
│ 门店打卡签到        库存自动扣减        现场收款/挂账     │
│ 代客下单补货        应收自动增加        打印收款小票       │
│      │                  │                  │             │
│      └──────────────────┴──────────────────┘             │
│                         ↓                                │
│ ⑦ 财务闭环：应收账龄管理 → 催款提醒 → 对账单分享 → 结清  │
└──────────────────────────────────────────────────────────┘
```

## 二、UI/UX 交互设计

### 2.1 主界面标签页（多窗口切换）

> 像浏览器标签页一样，用户可以同时打开多个功能窗口，快速切换。

```
┌─────────────────────────────────────────────────────────┐
│ [🏠 首页] [📋 销售订单 SO-...] [📦 库存查询] [👤 客户档案] │  [+] │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  当前激活的标签页内容区域                                   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**标签页规则：**
- 左侧导航点击 → 新标签页打开（不覆盖当前）
- 右键标签：关闭 / 关闭其他 / 关闭左侧 / 关闭右侧 / **固定标签**（仅图标）/ 刷新
- 底部状态栏「**最近关闭**」按钮，展开最近10条历史，一键恢复
- 已修改未保存的标签页显示圆点提示
- 最多同时打开10个标签页，超出提示关闭旧标签
- 标签页状态持久化：刷新/重新登录恢复上次打开标签
- 单据详情页自动以标签页打开

**技术：** Vue3 + Element Plus `el-tabs` (type="card")，Pinia 管理标签状态

### 2.2 侧边栏悬停展开

> 鼠标悬停在侧边栏图标上 → 自动展开子模块菜单，不需要点击。

```
默认状态（收起）：          悬停状态（展开）：
┌──┐                      ┌──────────────┐
│📋│                      │📋 销售管理     │
│  │                      │  ├ 销售订单    │
│📦│  鼠标移到📋 →         │  ├ 销售出库    │
│  │                      │  ├ 销售退货    │
│💰│                      │  └ 客户对账    │
│  │                      │              │
└──┘                      └──────────────┘
```

**交互细节：**
- 默认：图标模式（40px宽），只显示图标
- 鼠标悬停0.3秒后：右侧滑出浮层子菜单（200px宽），不挤占主内容区（`position: absolute`）
- 鼠标移出0.5秒后：自动收回（防误触发）
- 点击侧边栏顶部「图钉」按钮：锁定展开
- 移动端（触屏）：改为点击展开/收起，不支持悬停
- 键盘快捷键：`Alt+1` 销售、`Alt+2` 采购、`Alt+3` 仓库...

## 三、三端规划

### 3.1 PC桌面端（Electron）
**用户**：管理员、主管、财务、仓管
**定位**：全功能管理后台，标签页+悬停侧边栏

| 功能模块 | 说明 |
|---------|------|
| 系统初始化 | 公司信息/仓库/员工/权限/路线/打印模板/期初数据 |
| 档案管理 | 商品/客户/供应商档案 + 批量导入 |
| 采购管理 | 订单→入库→退货→发票→对账→统计 |
| 销售管理 | 订单→出库→退货→发票→对账→统计 |
| 仓库管理 | 实时库存/盘点/调拨/报损报溢/预警/分拣/装车 |
| 财务管理 | 收款/付款/预收预付/应收应付/发票/账户余额 |
| 报表分析 | 经营看板/销售报表/采购报表/库存报表/利润表 |
| CRM | 联系人/拜访记录/价格等级/流失预警 |
| 业务员管理 | 业绩统计/提成计算/拜访路线/外勤轨迹 |
| 辅助功能 | 数据备份/导入导出/操作日志/消息通知/打印管理 |

### 3.2 手机端（手机网页/H5）

> **核心理念**：同一个后端API，不同前端呈现。

#### 老板手机端（=PC端全功能适配）

**适配原则：**
- 所有数据接口和PC端完全一致
- 布局：卡片式代替表格，折叠菜单代替侧边栏
- 操作：长按=右键菜单，滑动=翻页
- 颜色语义统一：绿色=收入/完成，红色=支出/逾期，橙色=预警

#### 业务员手机端（8大简化窗口）

**核心诉求**：外勤操作，3步完成任何任务。

| 窗口 | 功能 | 说明 |
|------|------|------|
| 📋 待办订单 | 查看分配订单 | 🔴超时/🟡今日/🟢已处理，一键导航 |
| 🗺️ 拜访计划 | 按路线排序 | 基于路线权限+GPS距离排序 |
| 📸 门店打卡 | 定位+拍照+签到 | 200米内才能打卡（防作弊） |
| 🛒 快速下单 | 代客下单 | 扫条码/搜索商品/购物车/提交 |
| 🖨️ 打印小票 | 蓝牙打印 | 58mm/80mm热敏，销售单+收款凭证 |
| 📊 业绩查看 | 今日/本周/本月 | 销售额/回款/提成/排名 |
| 📦 查库存 | 多仓库存 | 客户等级价/上次进货/一键下单 |
| 💰 应收查询 | 客户欠款 | 账龄颜色/催款/明细 |

**设计原则**：界面极简、按钮大、字体大、支持离线、颜色统一

#### 仓管手机端

| 功能 | 操作流程 |
|------|----------|
| 扫码入库 | 扫条码 → 带出商品 → 填数量 → 确认 |
| 扫码出库 | 扫条码 → 确认 → 打印出库单 |
| 打印标签 | 蓝牙打印商品标签（名称/批次/保质期） |
| 库存查询 | 扫条码查实时库存，各仓分布 |

### 3.3 微信生态

#### 终端店老板H5下单

**下单流程：**
```
微信打开H5链接
    ↓
手机号验证（首次自动注册）
    ↓
商品列表（分类+搜索+等级价）
    ↓
加入购物车 → 结算（显示金额+欠款）
    ↓
确认下单 → 生成SO → 推送老板审核 → 通知业务员
    ↓
订单追踪：待审核→已审核→已出库→配送中→已送达
```

**技术方案：** Vue3 + Vant UI，手机号+验证码认证，分享链接带客户ID

#### Agent微信客服机器人

**三大场景：**

| 场景 | 方式 | 示例 |
|------|------|------|
| 📢 主动推销 | 朋友圈/群发/微信群 | "🔥冰红茶特价！整箱¥60！" |
| 🤖 智能接单 | 客户私聊下单 | "我要5箱冰红茶"→自动生成SO |
| 💬 智能客服 | 询价/查库存/查欠款 | 自动回复+状态查询 |

**安全边界：** 不修改价格 / 不修改库存 / 大额必审 / 删单退款转人工

| 场景 | 谁 | 打印什么 | 设备 | 纸张 |

### 4.2 PC端打印（A4）

- 模板：销售单/采购单/入库单/出库单/对账单/快递单
- 技术：jsPDF生成PDF → 浏览器打印API
- 支持：自定义Logo/批量打印/导出PDF

### 4.3 手机端蓝牙打印（58mm/80mm热敏）

| 规格 | 宽度 | 每行中文 | 适用场景 | 常见机型 |
|------|------|:---:|------|------|
| 58mm | 384点 | 16字 | 收款小票、简易订单 | 启锐QR-586、芯烨XP-58 |
| 80mm | 576点 | 24字 | 正式销售单（带明细表格） | 启锐QR-880、佳博GP-80 |

**技术方案：**
- Web Bluetooth API 连接 + ESC/POS 指令
- 小票模板：80mm完整版（商品明细表格）/ 58mm精简版（品名+数量+金额）
- 兼容：Web Bluetooth不支持时降级为生成图片→保存相册→手动打印

**打印设置面板：**

#### 1.4 路线权限系统

**数据结构关系：**

```
┌──────────┐      ┌──────────┐      ┌──────────┐
│  仓库     │──1:N──│  路线     │──N:M──│  业务员   │
│(warehouse)│     │ (route)  │     │(employee)│
└──────────┘      └──────────┘      └──────────┘
                        │
                      1:N
                        │
                  ┌──────────┐
                  │  客户     │
                  │(customer)│
                  └──────────┘
```

- **仓库↔路线**：一个仓库绑定多条路线（武昌总仓→武昌A线/B线/C线）
- **业务员↔路线**：一个业务员绑定多条路线（张三→武昌A线+武昌B线）
- **业务员↔仓库**：一个业务员绑定多个仓库
- **客户↔路线**：每个客户绑定一条路线+默认业务员+默认仓库

##### 客户档案（扩展字段）

| 字段 | 说明 |
|------|------|
| 绑定路线 | 该客户所属路线（必填） |
| 绑定业务员 | 默认负责业务员（可多个） |
| 默认仓库 | 发货默认仓库 |
| 客户等级 | VIP/A/B/C |
| 信用额度 | 超额提醒 |
| 绑定手机号 | 微信H5登录匹配 |
| 绑定微信 | 关联open_id |

##### 业务员权限规则

```
业务员A（绑定：武昌A线、武昌B线、武昌总仓）

✅ 可见：
  - 武昌A线+B线上的所有客户
  - 同路线其他业务员创建的销售单据（可查看不可修改）
  - 自己创建的销售单据（可查看可修改）
  - 武昌总仓的库存（只读）
  - 自己客户的收款记录+应收数据

❌ 不可见：
  - 汉口线上的客户
  - 其他路线业务员创建的销售单据
  - 其他仓库的库存
  - 采购单据
```

**单据可见性规则：**

### 模块三：采购管理

#### 3.1 采购订单
- 关联供应商，自动带出商品
- 数量/**进价**/金额自动计算
- **审核状态机：**

```
                    ┌── 审核通过 ──→ [已审核] → 可入库
                    │
[待审核] ──┤
                    ├── 驳回 ──→ [已驳回] → 作废（不可恢复）
                    │                  └→ 提供「复制单据」按钮
                    │
                    └── (免审角色) ──→ 自动[已审核]
```

- 驳回=作废，支持[复制单据]一键生成新单（新编号）
- 创建人角色勾选「采购单免审」→ 自动跳过审核

#### 3.2 采购入库
关联/不关联采购订单，扫码入库，**估价入库**（货先到票未到），确认→更新库存+应付+成本

#### 3.3 采购退货
关联原入库单，退货原因（质量/发错/过期），确认→库存减少+应付减少

#### 3.4 采购发票
关联入库单，专票/普票/收据，未收/已收/已认证

#### 3.5 供应商对账

```
搜索供应商 → 显示应付明细 → 勾选欠款单据 → 核销/付款
    ↓
期初应付 + 本期采购 - 本期退货 - 本期付款 = 期末应付
    ↓
点击「核销」→ 输入金额/方式 → 生成付款单PC
    ↓
打印对账单(A4) / 微信分享PDF
```

#### 3.6 采购返利
供应商/返利类型（年终/季度/单品）/返利金额/方式（抵扣/现金），关联采购记录

#### 3.7 采购统计
按供应商/商品/日期统计，趋势图，同比环比

### 模块四：销售管理

#### 4.1 销售订单
- 关联客户 → 自动带出售价、绑定路线、绑定业务员、默认仓库

#### 4.2 销售出库
关联/不关联订单，扫码出库，确认→库存减少+应收增加

#### 4.3 销售退货
关联原出库单，退货处理（退回入库/报损），确认→库存+应收减少

#### 4.4 销售发票
关联出库单，专票/普票/收据

#### 4.5 客户对账（核销流程）

```
搜索客户 → 显示应收明细 → 勾选欠款订单 → 核销/收款
    ↓
期初应收 + 本期销售 - 本期退货 - 本期收款 = 期末应收
    ↓

```

#### 4.6 销售统计

### 模块五：仓库管理

#### 5.1 实时库存
按仓库/商品查询，移动加权平均成本，库存流水，导出Excel

#### 5.2 库存预警
库存下限/上限/临期预警/滞销预警 → 微信+PC弹窗提醒

#### 5.3 库存盘点
全盘/分类/商品，账面vs实盘→自动差异，确认→生成盘盈入库或盘亏出库

#### 5.4 库存调拨
调出仓→调入仓，同仓校验，**默认需审核**（免审角色跳过），驳回=作废+复制

**调拨类型：**

| 类型 | 说明 |
|------|------|
| 仓库调拨 | 总仓 ↔ 分仓 |
| **车销调拨** | 总仓 → 业务员车仓 |
| 退货调拨 | 车仓 → 总仓 |

#### 5.5 报损报溢
报损（过期/破损）→库存减少，报溢（盘点多）→库存增加

#### 5.6 仓库分拣单 + 装车调度

```
分拣单：
- 关联销售出库单
- 按商品/客户分组拣货清单
- 拣货员确认拣货完成
- 支持PDA扫码拣货

装车调度：
- 按路线/客户排定发车顺序
- 关联车辆+司机
- 打印装车清单 + 车辆对货单
```

#### 5.7 车销支持（完整流程）

```
1. 装车出库
   仓管创建「车销出库单」→ 总仓→业务员车仓
   记录：业务员/车牌号/出门商品清单+数量

2. 手机开单
   业务员沿路线拜访 → 手机App开销售单
   → 现场收款 → 蓝牙打印小票

3. 回库报损
   回库清点剩余 → 系统对比：
     出门库存 - 销售开单 = 应有剩余
     实际剩余 vs 应有剩余
   差异 → 自动生成报损单（破损/丢失/少收）

4. 车销报表
   按业务员/车次统计：销售额/报损率/回款率
```

**新增单据类型：**

| 单据 | 前缀 | 说明 |
|------|:---:|------|
| 车销出库单 | VS | Vehicle Sales Out |
| 车销回库单 | VR | Vehicle Return |
| 车销报损单 | VL | Vehicle Loss |

**车仓概念：** 每个业务员绑定虚拟「车仓」，车销出库=总仓→车仓，回库=车仓→总仓

#### 5.8 窜货管理

**场景：** 业务员从蔡甸仓库拿货，卖到不属于自己路线的汉南区域。

**检测：** 业务员绑定路线 vs 销售单客户路线 → 不匹配标记"窜货"
**报表：** 窜货明细表（谁/什么商品/从哪到哪/金额）

### 模块六：财务管理

#### 6.1 收款登记

#### 6.2 付款登记
关联供应商+入库单（核销），现金/银行/微信/支付宝/支票，确认→应付减少

#### 6.3 预收款/预付款

| 类型 | 说明 | 使用方式 |
|------|------|---------|
| 预收款 | 客户先付款 | 销售出库时自动冲抵应收 |
| 预付款 | 公司先付供应商 | 采购入库时自动冲抵应付 |

```
预收款流程：
客户打款 → 财务录入预收款 → 预收余额增加
    ↓
客户下单 → 销售出库 → 自动从预收扣减
    ↓
预收不足 → 剩余计入应收

预付款流程：同上逻辑，冲抵应付
```

**自动抵扣：** `deduct = min(advance_balance, amount)`，优先用预收/预付冲抵

#### 6.4 账户余额
按账户（现金/银行/微信/支付宝）统计，收支明细流水

#### 6.5 应收应付管理

#### 6.6 发票管理
采购+销售发票，与入库/出库核销

#### 6.7 银行对账
导入CSV→自动匹配→人工核销

#### 6.8 费用合同 + 费用支出

```
费用合同：
- 合同编号/对方单位/费用类型（租金/广告/物流/其他）
- 合同金额/已付金额/未付金额
- 合同有效期/付款周期

费用支出：
- 关联费用合同（可选）
- 费用类型/金额/付款方式/收款方
- 自动记入收支流水
```

#### 6.9 赠品额度管理
每个商品设置赠品上限（占销售额% 或 绝对数量），超出提醒，赠品汇总报表

## 八、数据库核心设计

### 8.1 员工表 (employees)

```sql
CREATE TABLE employee (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    employee_no VARCHAR(20) UNIQUE NOT NULL,    -- 工号
    name VARCHAR(50) NOT NULL,                  -- 姓名
    phone VARCHAR(20) UNIQUE NOT NULL,          -- 手机号（登录账号）
    password_hash VARCHAR(255) NOT NULL,        -- 加密密码
    role VARCHAR(20) NOT NULL,                  -- admin/supervisor/sales/finance/warehouse
    warehouse_ids TEXT,                          -- 绑定仓库ID JSON数组
    route_ids TEXT,                              -- 绑定路线ID JSON数组
    bypass_audit TEXT,                           -- 免审核权限 {po:true, so:false, tf:false}
    online_status BOOLEAN DEFAULT 0,            -- 在线状态
    device_limit VARCHAR(100),                  -- 登录设备限制
    last_login_ip VARCHAR(50),
    last_login_time DATETIME,
    status VARCHAR(10) DEFAULT 'active',         -- active/inactive
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 8.2 路线表 (routes)

```sql
CREATE TABLE route (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    code VARCHAR(20) UNIQUE NOT NULL,            -- R001
    name VARCHAR(50) NOT NULL,                   -- 蔡甸城区路线
    warehouse_id INTEGER NOT NULL,               -- 所属仓库
    sort_order INTEGER DEFAULT 0,
    description TEXT,
    status VARCHAR(10) DEFAULT 'active',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 8.3 关联表

```sql
-- 仓库-路线关联
CREATE TABLE warehouse_route (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    warehouse_id INTEGER NOT NULL,
    route_id INTEGER NOT NULL,
    UNIQUE(warehouse_id, route_id)
);

-- 业务员-仓库关联
CREATE TABLE employee_warehouse (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    employee_id INTEGER NOT NULL,
    warehouse_id INTEGER NOT NULL,
    UNIQUE(employee_id, warehouse_id)
);

-- 业务员-路线关联
CREATE TABLE employee_route (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    employee_id INTEGER NOT NULL,
    route_id INTEGER NOT NULL,
    UNIQUE(employee_id, route_id)
);
```
